Overview

The GitHub Admin System is a lightweight, client-side admin panel that allows you to manage app details and download links by directly updating files in a GitHub repository. This system operates entirely in the browser using HTML, CSS, and JavaScript with no server-side processing required.

Prerequisites

Step 1: Create a GitHub Personal Access Token

  1. Go to GitHub Settings ‚Üí Developer settings ‚Üí Personal access tokens ‚Üí Tokens (classic)
  2. Click "Generate new token"
  3. Select the scopes required for your repository (typically "repo" for private repos, "public_repo" for public repos)
  4. Copy the generated token (you won't see it again)

Note: Store your token securely and never share it publicly.

Step 2: Identify Your Repository and Config File

You'll need:

  • Your GitHub repository name (format: owner/repository)
  • The path to your configuration file (e.g., apk_config.json)

Using the Admin System

Step 1: Access the Admin Panel

Navigate to github-admin.html in your browser

Step 2: Enter Authentication Details

  • Enter your GitHub Personal Access Token in the token field
  • Enter your repository name in the format "owner/repository"
  • Enter the path to your configuration file
  • (Optional) Check "Remember credentials" to store encrypted credentials locally
  • Click "Connect to GitHub"

Step 3: Manage App Details

Once connected, you can:

  • View current configuration
  • Edit app version, download URL, file path, and release notes
  • Validate the configuration before saving
  • Save updates directly to your GitHub repository

Security Best Practices

Important Security Notes

  • Never commit your GitHub token to version control
  • Use tokens with minimal required permissions
  • Regularly rotate your personal access tokens
  • Be cautious when using "Remember credentials" on shared computers
  • Log out when finished using the admin panel

Security Tips

  • Tokens are stored locally using basic obfuscation (not strong encryption)
  • Sensitive data is cleared from memory on page unload
  • All API calls are made directly from the browser to GitHub
  • No server stores your credentials

Configuration File Format

The system expects a JSON configuration file with the following structure:

{ "latest_version": "1.0.0", "latest_version_code": 1, "download_url": "https://example.com/app.apk", "file_path": "app.apk", "file_size": 1048576, "release_notes": "Initial release", "last_updated": "2023-01-01T00:00:00.000Z" }

Troubleshooting

Common Issues:

  • Authentication Failed: Verify your token has correct permissions and hasn't expired
  • File Not Found: Check the file path is correct and the file exists in the repository
  • CORS Error: Ensure GitHub's API endpoints are accessible from your domain
  • Network Error: Check your internet connection and firewall settings
